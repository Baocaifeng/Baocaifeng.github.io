    Linux

    This post indicates the possible steps or problems that I have met when install Debian on my old computer. It’s not a beginning tutorial with too many details. If you have no experience of any other linux os installation, you better find other tutorials in case to wast more time. If you have already some experience of linux os installation like Ubunto, Mint, Kali, whatever, and you have also an old computer that you don’t want to throught it away, then this post may help you a little.

    Before starting

    You should have:

    • An old computer
    • A new computer with stable internet connection
    • A USB disk which has more the 4GB space
    • Ready to solve any kind of problems you may have

    Install Linux OS (Debian)

    • Download debian iso from Debian

    • Insert your USB disk

    • If don’t have, install rufus then open it:

    • write iso image to your USB disk

    • change your BIOS setting, make sure it allowed to boot from USB disk

    • choose graphic install

    • if your debian reports that your computer doesn’t support DHCP protocal, configure your IP address manually.

    • if your debian reports that you lack some no-free firmware drivers, this happens to old computers, Download what you need

    • configure your disk partition manually

    • set mount point for each partition or simply use root point

    Set up network

    • make sure your net card driver is installed
    # dmesg | grep -i wlan
# dmesg | grep -i wireless
    • list network interfaces:

    #ip a

    • Sample /etc/network/interface Config File
    ># The loopback network interface
>auto lo
>iface lo inet loopback
># The primary network interface
>auto eth0
>iface eth0 inet static
>        address 192.168.1.5
>        netmask 255.255.255.0
>        network 192.168.1.0
>        gateway 192.168.1.1
 
># The wireless network interface with dhcp
>auto wlan0
>iface wlan0 inet dhcp
>    wpa-ssid nixcraft
>    wpa-key-mgmt WPA-PSK
>    wpa-group TKIP CCMP
>    wpa-psk YOYR-PASSWORD-HERE
    • restart network service
    > /etc/init.d/networking restart

    • set sshd service for remote connection

    • update intel microcode, download intel microcode driver here https://pkgs.org/download/intel-microcode

    • check your debian version

    lsb_release -a

    • for debian 10 (buster) amd64 https://debian.pkgs.org/10/debian-nonfree-amd64/intel-microcode_3.20200616.1~deb10u1_amd64.deb.html

    • If you don’t have internet in the old computer beacause some problems/errors you can’t solve, you can download this .deb package

    • from your new computer and use a USB device

    • Instert the USB which contains what you need to your old computer (debian)

    • the check the device information

    ls /dev | grep sd

    for me, the sdax is the harddisk, sdbx is the USB device, you may have something similar, x is usually a number

    • Mount USB device, x is a number, ex: 1

    mkdir /mnt/usb mount /dev/sdbx /mnt/usb ls /mnt/usb/ then you can see all contents in the USB device, here is a beginner tutorial for Debian USB https://wiki.debian.org/USBDrive

    • umount USB

    umount /mnt/usb

    • install .deb package from file

    apt install -f xxxx.deb

    • Debian may reports the dependencies that you need, go to https://debian.pkgs.org/10/debian-nonfree-amd64/intel-microcode_3.20200616.1~deb10u1_amd64.deb.html and find the required dependencies and download

    • intel realtek firmware (for voice card) https://packages.debian.org/buster/all/firmware-realtek, if you need some no free firmware and you have the internect connection, probably it’s better to add no-free firmware to your source list and update your os but if you don’t have internet connection, and you know what kind of firmware you need, you can download them and install separately

    • intel iwlwifi firmware (for wifi, bluetooth, etc) https://packages.debian.org/buster/all/firmware-iwlwifi/download

    • after installing iwlwifi firmware, you can connet to the internet

    • scan wifi entry point (ssid), remember to change the name of your wireless card

    iwlist wlp8s0 scan | grep SS

    • transfer file from windows powershell (new computer) to debian linux (old computer), use scp command

    scp your_file.txt remote_linux_username@linux_hostname_or_local_ip:/remote/directory/on/linux scp -r ‘C:\path_to_your_folder’ remote_linux_username@linux_hostname_or_local_ip:/remote/directory/on/linux/

    • down load Apache Hadoop https://hadoop.apache.org/releases.html, in my case, I choose the 3.2.2 version (stable)

    • install hadoop, I use the single node cluster cause I have only one old computer. https://hadoop.apache.org/docs/stable/hadoop-project-dist/hadoop-common/SingleCluster.html

    • beacause Apache Hadoop is based on JVM, you need to install Java, choose the right java version here https://cwiki.apache.org/confluence/display/HADOOP/Hadoop+Java+Versions In my case, I use java 8

    • Dowload Debian openjdk 8 package here jre: https://packages.debian.org/stretch/amd64/openjdk-8-jre/download jdk: https://packages.debian.org/stretch/amd64/openjdk-8-jdk/download you may also need jdk-headless / libxt-dev packages

    • If you want to install Hadoop as cluster, you need dowload pdsh and ssh if you don’t have it https://packages.debian.org/stretch/amd64/pdsh/download Hadoop need a good connection between nodes If you just use hadoop in the single cluster, it’s ok if you don’t install ssh or pdsh, because hadoop doesn’t need communicate between machines

    • congigure Debian

    • https://adoptopenjdk.net/installation.html?variant=openjdk8&jvmVariant=hotspot#x64_linux-jre a good website for installing java 8 on Debian 10 (bluser)

    • change the ownership of files

    chown [owner/group owner] [file name] chown caifeng yourfile chown :caifeng yourfile chown root:caifeng yourfile

    • configure java, hadoop path export JAVA_HOME=/home/caifeng/.java8/jdk8u282-b08
      export PATH=$PATH:/home/caifeng/.java8/jdk8u282-b08/bin:/home/caifeng/.java8/jdk8u282-b08-jre/bin
      export PATH=$PATH:/home/caifeng/.hadoop/hadoop-3.2.2/bin

    • check if hadoop works

    $ hadoop

    • Standalone Operation By default, Hadoop is configured to run in a non-distributed mode, as a single Java process. This is useful for debugging.
